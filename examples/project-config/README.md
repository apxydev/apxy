# APXY Project Configuration

This directory contains full example `config.json` and `config.local.json` files for APXY's project-scoped configuration system.

## Quick Start

```bash
# Initialize .apxy/ in your project
apxy init

# Validate before starting
apxy config validate

# Start proxy — auto-discovers .apxy/config.json
apxy proxy start
```

## How It Works

1. Create `.apxy/config.json` in your project root (commit to git — shared with team)
2. Optionally create `.apxy/config.local.json` for personal overrides (gitignored)
3. Run `apxy proxy start` — it auto-discovers and loads your config

The config is imported into APXY's global SQLite database on every startup,
overwriting previous state so the config file is always the source of truth.

## Config Precedence (highest wins)

```
CLI flags
    ↓
.apxy/config.local.json  (personal, gitignored)
    ↓
.apxy/config.json        (shared, committed)
    ↓
~/.apxy/config.json      (global user defaults)
    ↓
Built-in defaults
```

## File Structure

```
your-repo/
  .apxy/
    config.json           # Committed to git — shared team config
    config.local.json     # Gitignored — personal overrides
    .gitignore            # Auto-generated by 'apxy init'
```

## Field Reference

### `proxy` section — proxy startup defaults

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `port` | int | 8080 | Proxy listen port (1–65535) |
| `control_port` | int | 0 (port+1) | Control API port (0 = auto) |
| `web_port` | int | -1 (port+2) | Web UI port (-1 = auto, 0 = disabled) |
| `max_body` | int | 1048576 | Max body capture in bytes |
| `mitm_all` | bool | false | MITM all HTTPS (full deep inspection) |
| `bypass_domains` | string[] | [] | Domains to tunnel without MITM |
| `upstream_proxy` | string | "" | Upstream proxy URL for chaining |
| `auto_validate` | bool | false | Auto-validate traffic against imported schemas |
| `no_system_proxy` | bool | false | Skip OS proxy configuration |
| `no_mdns` | bool | false | Skip mDNS/Bonjour advertisement |

### `ssl_domains` — string[]

Domains to enable HTTPS interception for. Supports wildcards (`*.stripe.com`).
These are merged into the proxy's SSL host list on startup.

### `mock_rules` — array

Mock API responses before they reach the real server.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Unique identifier |
| `name` | string | yes | Display name |
| `priority` | int | no | Higher = matched first (default 0) |
| `active` | bool | no | Enable/disable (default true) |
| `url_pattern` | string | yes | URL to match |
| `match_type` | `"exact"` \| `"wildcard"` \| `"regex"` | yes | How to match url_pattern |
| `method` | string | no | HTTP method filter (empty = all) |
| `header_conditions` | map | no | Request headers that must match |
| `response_status` | int | yes | HTTP response status code (100–599) |
| `response_headers` | map | no | Response headers to send |
| `response_body` | string | no | Response body string |
| `delay_ms` | int | no | Delay before responding (ms, ≥ 0) |

### `filter_rules` — array

Capture include/exclude rules. Block rules reject matching hosts. Allow rules
create a whitelist — only matching hosts are captured.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Unique identifier |
| `type` | `"allow"` \| `"block"` | yes | Filter action |
| `target` | string | yes | Domain or URL pattern (wildcards supported) |
| `active` | bool | no | Enable/disable (default true) |

### `redirect_rules` — array

Redirect traffic from one URL pattern to another.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Unique identifier |
| `name` | string | yes | Display name |
| `from_url` | string | yes | Source URL pattern |
| `to_url` | string | yes | Destination URL |
| `match_type` | `"exact"` \| `"wildcard"` \| `"regex"` | yes | Match mode |
| `active` | bool | no | Enable/disable (default true) |

### `scripts` — array

JavaScript scripts that execute on matching traffic.
**Security**: scripts require explicit trust confirmation on first use.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Unique identifier |
| `name` | string | yes | Display name |
| `active` | bool | no | Enable/disable (default true) |
| `code` | string | yes | JavaScript code |
| `match_expr` | string | no | DSL match expression |
| `hook` | `"onRequest"` \| `"onResponse"` | yes | When to run |

### `breakpoint_rules` — array

Pause matching traffic for inspection and modification.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Unique identifier |
| `name` | string | yes | Display name |
| `active` | bool | no | Enable/disable (default true) |
| `match_expr` | string | yes | DSL match expression |
| `phase` | `"request"` \| `"response"` \| `"both"` | yes | When to pause |
| `timeout_ms` | int | no | Auto-resume timeout (default 30000) |

### `network_condition` — object

Simulate degraded network conditions.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `latency_ms` | int | 0 | Added latency per request (ms) |
| `bandwidth_kbps` | int | 0 | Bandwidth limit in kbps (0 = unlimited) |
| `packet_loss_pct` | int | 0 | Packet loss percentage (0–100) |

### `no_caching` — object

Force cache-busting headers.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | false | Enable cache busting |
| `host` | string | "" | Restrict to specific host (empty = all) |

### `custom_cas` — array

Per-domain custom Certificate Authorities with inline PEM.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `domain_pattern` | string | yes | Domain glob (e.g. `*.internal.corp`) |
| `label` | string | no | Display label |
| `cert_pem` | string | yes | PEM-encoded certificate |
| `key_pem` | string | yes | PEM-encoded private key |

## config.local.json — Personal Overrides

`config.local.json` is auto-gitignored. Use it for developer-specific settings:

- Different port (avoid conflicts with other local services)
- `no_system_proxy: true` (when using curl's `-x` flag directly)
- Network conditions for testing (simulate slow connections)
- Personal SSL domains (e.g. local dev services)

Any field present in `config.local.json` **replaces** the same field from `config.json`.
Array fields that are present (even as `[]`) replace the base array entirely.

## Team Workflow

1. Developer A: `apxy init` → edits `config.json` with team rules → commits
2. Developer B: `git pull` → `apxy proxy start` → config auto-loaded
3. Developer B: adds `config.local.json` for personal overrides (gitignored)
4. Runtime changes via Web UI: use `apxy config export --project` to save back to `config.json`

## Validation

```bash
# Validate current project config
apxy config validate

# Validate a specific file (AI-friendly JSON output)
apxy config validate --file .apxy/config.json --format json
```

Exit code 0 = valid, 1 = errors.

## Hot-Reload

APXY watches `config.json` and `config.local.json` for changes while the proxy is running.
Rule sections (mock rules, scripts, etc.) are reloaded automatically on save.
Proxy settings (port, mitm_all) require a restart.

Use `--no-watch` to disable hot-reload.
